☂️ Section 1: 情報整理（空・雨・傘）

📝 1-1. 演習用ダミーメモ
コピー用テキスト

「事実」と「感想」が混ざったダメな議事録のサンプルです。これを整理してみましょう。

【会議メモ：メンター制度振り返り】 ・今年のメンター制度、なんとなく盛り上がりに欠ける気がする。 ・アンケートの満足度は5点満点中3.2点だった。去年は3.8点。 ・メンター（先輩）側から「業務が忙しくて面談時間が取れない」という声が3件あった。 ・やっぱり若手もリモートワークが増えてるから、関係性が作りにくいんじゃないか？ ・他社の事例だと、ランチ代を補助すると会話が増えるらしいよ。 ・メンティー（若手）の離職率は今のところ横ばい。 ・制度自体を廃止するのは違うと思うけど、形骸化してるなら変える必要あり。 ・人事部の担当者が来月から1名減る予定。 ・とりあえず、もっとメンターに「ちゃんとやって」ってアナウンスしようか。
✨ 1-2. 空・雨・傘プロンプト
整理・構造化
# 命令 以下の【テキスト】を読み込み、マッキンゼーの「空・雨・傘」のフレームワークを用いて情報を整理・構造化してください。 # 制約条件 1. **空（事実）：** テキストに書かれている客観的な事実・データのみを抜き出すこと。ここにあなたの意見や推測を入れてはいけません。 2. **雨（解釈）：** 「空」の事実から読み取れる真の課題、リスク、または洞察を記述すること。 3. **傘（行動）：** 「雨」の解釈に基づき、具体的に取るべきアクションや提案を記述すること。 # テキスト [ここにさっきの「ダミーメモ」を貼り付けてください]

🧠 Section 2: 思考の深掘り

🧙‍♂️ 2-1. 3人の賢者 (Step 1)
発散・多角化
# 命令 あなたは優秀なビジネス参謀です。以下の【課題】について、異なる3つの視点（ロール）から議論を行い、解決策のアイデアを出してください。 # 3つの視点 1. **経営戦略家：** コスト対効果（ROI）と長期的利益を最優先する視点 2. **現場リーダー：** 実行可能性と現場メンバーの負担軽減を最優先する視点 3. **顧客心理の専門家：** ユーザー（または従業員）の感情と満足度を最優先する視点 # 課題 [ここにあなたの解決したい課題を入力] # 出力形式 それぞれの視点からの意見と、それらを統合した初期アイデアを出力してください。

📐 Section 3: 物語の構成（S-C-Q-A）

📊 3. S-C-Q-Aプロンプト
提案・ストーリー
# 命令 これまでの検討内容をもとに、上司に提案するための資料構成案を作成してください。 「S-C-Q-A」モデルを使用し、読み手が自然と納得するストーリーにしてください。 # S-C-Q-Aの定義 * **S (Situation - 状況)：** 読み手が既に知っており、合意できる背景事実。 * **C (Complication - 複雑化)：** Sの中で起きている問題、変化、困りごと。 * **Q (Question - 問い)：** Cを解決するために、今答えるべき論点。 * **A (Answer - 答え)：** Qに対する核心的な解決策（今回の提案）。 # 出力形式 スライドの構成案として出力してください。
